Experience

Enlighten.Net

Back-End Software Developer

Time-Frame: June - August 2020

I had returned during the Summer of 2020 to continue my work with Enlighten.Net. Development was mainly done using C# and the .NET framework. We managed our projects using DevOps and the Visual Studio Community IDE. I worked on their ENet Docs application. Here are some of the things that I worked on during the summer!

  • First, I continued my initial implementation on the main enhancement I worked on last summer, which was to add a storage provider which would programatically upload storage items to the Amazon S3 Bucket. I changed the initial provider to include storage items from Enlighten.Net
  • Then, I updated their configuration system to further customize our Amazon S3 storage provider. Using the Windows Registry, I set up the Amazon S3 Bucket Configuration within the customer's configuration, allowing them to use one of Enlighten.Net's four buckets (which were distributed among the four Availability Zones throughout the US) and, if they have one, use one of their own buckets. I also added the configuration in which the user can have a default bucket, or they can specify a bucket to store each type of file (for example they can have a default bucket where everything is stored, but can specify a bucket for just their PDF files).
  • Next, I resolved a couple of bugs within their Web API. Specifically, I provided a solution for their 'Save' button, which at times was enabled when it should not have been. I also added functionality to a 'Sign and Move' button, in which if the user had permission to upload a file, they would automatically give permission to save the file (usually, the person would have to sign their request before saving it).
  • Finally, I began my implementation for a VerifyBucket tool. This command-line tool, given the company ID, would synchronize our database with the data that is stored in each Amazon bucket they have listed and report any differences between our database and the data in their Amazon Bucket/s.

To learn more about Enlighten.Net, please visit: https://www.enlighten.net/

Enlighten.Net

Back-End Software Developer

Time-Frame: June - August 2019

During the summer of 2019, I interned at Enlighten.Net as a back-end software developer. Development was mainly done using C# and the .NET framework. We managed our projects using DevOps and the Visual Studio Community IDE. I worked on their ENet Docs application. Here are some of the things that I worked on during the summer!

  • One of the main issues that Enlighten.Net was facing was the efficient storage of their documents. ENet Docs stores millions of files, and while Enlighten.Net was able to store these documents on their on-site servers, they frequently had to add hard drives, and managing their servers were becoming increasingly difficult. Therefore, they have decided to move to storing incoming documents on the cloud. After some research, they decided that Amazon S3 would best suit their needs. I was primarily responsible for researching Amazon S3, analyzing whether it truly fit the needs of Enlighten.Net, and looking for any pitfalls that could potentially harm the ENet Docs product. I then implemented a simple storage provider that allows the application to upload, stream, or delete documents stored on Amazon S3.
  • Additionally, I began to organize ENet Doc's configuration settings into a JSON file, and learned the basics of Web APIs.

To learn more about Enlighten.Net, please visit: https://www.enlighten.net/
